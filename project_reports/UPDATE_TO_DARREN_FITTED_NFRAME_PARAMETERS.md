# Update To Darren: Fitted N-Frame Parameters

Date: 2026-06-08

## What we did

We used real CMS data only. We matched high-boundary events to nearby controls from the same CMS context, then fitted N-Frame boundary parameters from the case-control differences.

## Did the boundary survive matching?

Yes. The boundary survived quality cleaning and matched controls. The result is strongest for the hand-defined boundary tails and weaker, but still present, for the unsupervised tails.

## Fitted parameters

The fitted weights are: P_displacement_proxy=0.357, P_reconstruction=0.211, P_multiplicity=0.202, P_btag_structure=0.093, P_visible_energy=0.073, P_missing=0.059, P_compression=0.006.

## Strongest parameters

The strongest parameters are the secondary-vertex/displacement proxy, reconstruction complexity, and multiplicity. B-tag structure, visible energy and missing energy also contribute.

## Weak parameter

Compression-like imbalance is weak in the fitted equation, so it should be treated as secondary for now.

## SUSY question

This is not evidence that SUSY was found. The strongest honest claim is that we have fitted a real-data N-Frame boundary-stress equation that survives trigger, quality and matched-control checks.

## Next step

Apply this fitted boundary equation to independent real CMS samples/runs and manually inspect the top fitted-boundary events.
